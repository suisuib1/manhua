# Figma 原型图生成提示词

请根据以下要求，为微信小程序“日记生成漫画”生成一套移动端 MVP 原型图。请输出适合 Figma 设计文件的多页面原型，页面尺寸优先使用 iPhone 14/15 规格，宽度按 375px 移动端设计。

## 项目名称

日记生成漫画

## 产品定位

这是一款私人日记漫画本微信小程序。用户每天输入日记并上传照片，系统根据日记、照片、角色档案和情绪标签生成 Q 版漫画章节，最终形成一本只属于用户自己的个人漫画书。

初版只做个人使用，不做公开社区、不做评论、不做点赞、不做关注、不做公开广场。

## 目标用户

- 喜欢记录生活、写日记、保存回忆的普通用户。
- 希望把照片和文字变成温暖漫画故事的用户。
- 重视隐私，只想自己查看或私密分享某个章节的用户。

## 小程序端设计要求

- 按微信小程序交互习惯设计：顶部导航清晰，主要操作按钮明显，底部可使用轻量 tab 或固定入口。
- 页面触控区域要足够大，主要按钮适合单手操作。
- 信息密度低，不要拥挤。
- 所有页面都要体现“私人、温暖、治愈、可爱”的氛围。
- 不出现社区、评论、点赞、关注、排行榜、公开广场等设计。
- 不出现支付、充值、会员、订单、余额、支付回调等设计。
- 不出现后台管理端、运营后台、审核后台等设计。
- 不要展示完整原始日记给分享访问者，分享页只展示用户允许分享的章节内容。

## 视觉风格

- 整体风格：温馨、治愈、可爱、Q 版漫画感。
- 页面氛围：像一本个人漫画日记本，轻松、柔和、有记忆感。
- 主色建议：奶油白、浅橙、浅粉、浅蓝，可以搭配少量柔和绿色作为点缀。
- 卡片：圆角、大留白、轻阴影，像贴纸、便签或漫画书页。
- 图标：圆润、可爱、低复杂度。
- 按钮：清晰、柔和、适合微信小程序触控。
- 字体层级：标题清晰，正文温和，辅助文字偏浅。
- 漫画主题：使用 Q 版漫画分镜、章节封面、气泡、贴纸感元素，但不要过度装饰。
- 隐私感：避免公开社交氛围，强调“我的漫画书”“私密章节”“仅链接可见”。

## 页面清单与核心模块

### 1. 首页 / 漫画书首页

目标：展示用户的默认漫画书和历史章节入口。

核心模块：

- 顶部欢迎语，例如“我的漫画日记本”。
- 默认漫画书封面区域。
- 今日创建入口按钮，例如“写今天的漫画日记”。
- 最近章节列表，展示章节标题、日期、生成状态、缩略图。
- 免费次数提示，例如“今日/剩余可生成次数”。
- 角色档案入口。
- 我的/设置入口。

状态设计：

- 空状态：还没有章节时，引导用户创建第一章。
- 生成中状态：章节卡片显示生成中。
- 失败状态：章节卡片显示重试入口。

### 2. 创建日记章节页

目标：让用户填写章节基础信息并开始创建。

核心模块：

- 章节标题输入。
- 日期显示，允许同一天多章。
- 页数选择：随机生成、自定义 1-8 张。
- 情绪/风格标签选择：温馨、搞笑、治愈、童话、可爱、日常、开心、难过、纪念。
- 下一步按钮：进入日记和照片填写。

设计注意：

- 标签只能体现 Q 版大风格下的微调，不要出现写实、赛博、恐怖等风格。
- 文案要轻柔，不要有复杂设置感。

### 3. 上传照片与填写日记页

目标：提交生成漫画所需的文字和照片。

核心模块：

- 日记输入框，提示用户记录今天发生的事。
- 照片上传区，支持 1-9 张照片。
- 已上传照片网格，展示排序和删除入口。
- 隐私提示：日记原文不会公开，分享页不会展示未授权隐私内容。
- 生成前检查提示：内容将用于生成漫画。
- 提交生成按钮。

状态设计：

- 未上传照片时展示温柔引导。
- 照片数量不足或超过限制时展示轻提示。
- 日记为空时展示轻提示。

### 4. 角色档案页

目标：帮助用户维护固定主角形象，让后续章节尽量保持角色一致。

核心模块：

- 主角昵称。
- 身份或关系，例如“我”“女儿”“朋友”等。
- 年龄段或形象年龄。
- 发型、发色、脸型、常见表情。
- 常穿服装或标志性配饰。
- 性格关键词。
- 参考头像/参考照片入口。
- 保存按钮。

设计注意：

- 页面要像“主角设定卡”。
- 明确表达角色档案会帮助漫画保持人物一致。
- 不要表现复杂训练、模型微调或高级 AI 参数。

### 5. 漫画生成任务等待页

目标：展示异步生成过程，让用户理解当前状态。

核心模块：

- Q 版漫画生成中的插画或分镜占位。
- 状态文字：排队中、生成中、整理漫画页。
- 进度感设计，可以是步骤条或轻量动画占位。
- 当前章节标题。
- 预计等待提示。
- 返回漫画书按钮。

状态设计：

- PENDING：等待开始。
- RUNNING：生成中。
- FAILED：生成失败，展示失败原因摘要和免费重试按钮。
- SUCCEEDED：生成完成，进入章节详情。

### 6. 章节详情页

目标：展示生成完成的漫画章节。

核心模块：

- 章节标题、日期。
- 漫画图片列表，每张图表现为多格漫画分镜。
- 角色一致性和 Q 版风格要在视觉上统一。
- 操作按钮：分享章节、重新生成、删除章节。
- 轻量隐私提示。

设计注意：

- 不展示完整原始日记正文作为默认内容。
- 重点展示漫画结果和章节氛围。
- 删除与重新生成应避免误触，可用二次确认弹层。

### 7. 章节分享页

目标：给未登录访问者通过私密链接查看被授权章节。

核心模块：

- 分享章节标题。
- 章节漫画图片。
- “来自某位用户的私密漫画章节”一类温和说明。
- 隐私提示：仅链接可见。
- 可选的小程序打开/返回入口。

严格限制：

- 不展示用户未授权的原始日记全文。
- 不展示其他章节。
- 不展示整本漫画书。
- 不提供评论、点赞、关注、转发到公开广场等社交模块。
- 不设计搜索引擎索引或公开访问入口。

### 8. 免费次数提示页 / 生成次数不足提示状态

目标：当免费次数不足时，温和提示用户暂不可生成。

核心模块：

- 当前剩余免费次数。
- 次数不足提示。
- 返回漫画书或稍后再试按钮。
- 规则说明：初版只做免费次数，不提供充值或会员。

严格限制：

- 不出现充值按钮。
- 不出现会员套餐。
- 不出现订单、余额、支付入口。

### 9. 我的页面 / 设置入口

目标：提供个人入口和基础设置。

核心模块：

- 微信头像和昵称区域。
- 默认漫画书入口。
- 角色档案入口。
- 免费次数查看。
- 隐私说明入口。
- 关于产品入口。

设计注意：

- 不做后台管理入口。
- 不做钱包、会员、订单入口。
- 不做社区个人主页。

## 核心用户流程

请在 Figma 中体现以下流程关系：

1. 用户进入首页 / 漫画书首页。
2. 用户点击“写今天的漫画日记”。
3. 用户填写章节标题、选择页数和情绪标签。
4. 用户输入日记并上传 1-9 张照片。
5. 用户提交后进入生成任务等待页。
6. 生成成功后进入章节详情页。
7. 生成失败时允许免费重试。
8. 用户可在章节详情页分享章节私密链接。
9. 分享访问者只能看到该章节分享页。
10. 用户可进入角色档案页维护固定主角。
11. 用户可在我的页面查看设置、角色档案和免费次数。

## 原型输出要求

- 生成完整 MVP 页面原型，不要生成营销落地页。
- 页面之间要有清晰跳转关系。
- 用真实中文界面文案，不要用 lorem ipsum。
- 重点体现“日记变漫画”“私人漫画书”“Q 版治愈漫画”“章节私密分享”。
- 所有页面按移动端优先设计，宽度 375px。
- 可以使用柔和插画占位、漫画分镜占位、贴纸元素和手账感装饰。
- 不要设计非 MVP 功能。
