# MVP 范围

## 第一版只做

- 微信登录。
- 默认漫画书。
- 角色档案。
- 日记章节创建。
- 照片上传记录。
- 异步生成任务。
- Q 版漫画生成。
- 章节查看。
- 章节删除。
- 章节重新生成。
- 章节私密分享。
- 免费次数。
- 简单敏感词过滤。

## 明确不做

- 不做公开社区。
- 不做评论、点赞、关注。
- 不做整本书分享。
- 不做单张漫画图独立分享。
- 不做支付、订单、充值、会员、余额、回调。
- 不做后台管理系统。
- 不绑定具体 AI 供应商。
- 不做复杂内容审核平台。
- 不做复杂角色训练。
- 不创建微信小程序工程。
- 不创建 Java 后端工程。
- 不创建数据库 migration。

## MVP 判断标准

任何新增需求进入开发前，必须先判断是否服务于“私人日记漫画本”的第一版闭环：

1. 用户能登录。
2. 用户能维护固定主角角色档案。
3. 用户能用日记和照片创建章节。
4. 系统能异步生成 Q 版漫画。
5. 用户能查看、删除、重新生成章节。
6. 用户能私密分享单个章节。
7. 系统能控制免费次数和基础内容安全。

不满足以上闭环的需求，默认不进入 MVP。
