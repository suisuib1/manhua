# Figma Skills 使用说明

本文档说明后续在 Codex 中如何使用 Figma 相关 skills。所有 Figma 相关任务仍必须遵守 AGENTS.md：一次只推进一个小目标，最小改动，验证通过后才允许中文 commit，不自动 push。

## 1. use_figma

用途：在任何 Figma 相关任务前先加载规则，尤其是在调用 Figma MCP、读取设计稿、修改 Figma 文件或准备设计转代码前。

适用场景：

- 准备生成 Figma 原型提示词。
- 准备读取 Figma 设计稿。
- 准备分析 Figma 页面、节点、组件、变量。
- 准备设计转代码前确认 Figma 操作规则。

示例命令：

```text
Use the use_figma skill.
```

注意：

- 如果只是写文档提示词，可以只读取需求文档并生成提示词，不需要直接调用 Figma MCP。
- 如果要实际操作 Figma 文件，必须先加载 use_figma 规则。

## 2. Figma

用途：连接 Figma MCP，读取或分析 Figma 文件、页面、节点、组件、颜色、间距和交互状态。

适用场景：

- 用户已经提供 Figma 文件链接。
- 用户已经提供 Figma 节点 ID。
- 需要 Codex 读取设计稿结构。
- 需要总结页面、组件、颜色、字体、间距、状态和交互。

示例命令：

```text
Use the Figma skill. 请读取这个 Figma 文件并总结页面结构、组件、颜色、间距和交互状态。
```

注意：

- 读取设计稿时优先只读分析。
- 不要在没有用户确认的情况下改 Figma 文件。
- 如果设计稿信息不足，应先输出问题，不要猜测实现。

## 3. Figma Implement Design

用途：把已经确认的 Figma 设计稿转成项目代码。

适用场景：

- 原型已经由用户确认。
- 已经明确要实现哪个页面或哪个小模块。
- 已经知道目标工程位置和允许修改范围。

示例命令：

```text
Use the Figma Implement Design skill. 请只实现“创建日记章节页”，不要改其他页面。
```

注意：

- 一次只实现一个页面或一个小模块。
- 必须遵守 AGENTS.md 的最小改动规则。
- 不允许顺手重构其他页面。
- 不允许扩大 MVP 范围。
- 实现前必须明确允许修改范围、禁止修改范围、验证命令和是否允许 commit。

## 推荐阶段顺序

### 第一步：整理原型提示词

使用 use_figma，读取规则，基于 PRD 和 MVP 文档生成或整理 Figma 原型提示词。

当前项目可直接使用：

```text
docs/FIGMA_PROTOTYPE_PROMPT.md
```

### 第二步：在 Figma 中生成原型图

用户将 docs/FIGMA_PROTOTYPE_PROMPT.md 复制到 Figma AI、FigJam 或设计生成工具中，生成小程序 MVP 原型。

### 第三步：用户人工确认原型图

用户检查页面范围、视觉风格、隐私边界和交互流程，确认是否符合 PRD。

重点检查：

- 是否只覆盖 MVP。
- 是否是私人日记漫画本。
- 是否只做章节私密分享。
- 是否没有社区、评论、点赞、关注。
- 是否没有支付、充值、会员。
- 是否没有后台管理端。

### 第四步：读取已确认的 Figma 设计稿

使用 Figma skill 读取已确认的 Figma 文件，整理页面结构、组件、颜色、间距和交互状态。

### 第五步：设计转代码

使用 Figma Implement Design skill，一次只实现一个页面，落地到微信小程序代码。

推荐顺序：

1. 首页 / 漫画书首页。
2. 创建日记章节页。
3. 上传照片与填写日记页。
4. 角色档案页。
5. 漫画生成任务等待页。
6. 章节详情页。
7. 章节分享页。
8. 免费次数提示状态。
9. 我的页面 / 设置入口。

每次实现前都必须重新确认本轮小目标，不能一次性实现全部页面。
