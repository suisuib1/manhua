# UI 开发任务拆分

本文档将 UI 参考图拆分为后续可逐个开发的小任务。每个任务必须单独执行、单独验证、验证通过后中文 commit，不允许自动 push。

## 总体原则

- 只做微信小程序 MVP 页面。
- 前端先使用 mock 数据和占位状态。
- 不接真实 AI 服务。
- 不接 Java 后端接口。
- 不创建数据库文件。
- 不引入依赖。
- 不做社区、评论、点赞、关注、公开广场。
- 不做支付、充值、会员、订单、余额。
- 不扩大 PRD 范围。

## Issue 1：小程序基础导航和页面骨架确认

目标：

- 确认当前 manhua-xcx/ 的页面结构。
- 规划 MVP 页面路径、底部导航和页面跳转关系。
- 处理默认 pages/index 和 pages/logs 与 MVP 页面的关系。

允许修改文件：

- manhua-xcx/app.json
- manhua-xcx/app.wxss
- manhua-xcx/pages/ 下必要页面目录
- 可新增 manhua-xcx/utils/mock.js 或类似 mock 文件

禁止修改内容：

- 禁止开发完整页面业务逻辑。
- 禁止接入后端接口。
- 禁止删除或移动 manhua-xcx/。
- 禁止修改 project.config.json。
- 禁止引入依赖。

验证方式：

- 使用微信开发者工具打开 manhua-xcx/，确认页面路径有效。
- 检查 app.json 页面配置无语法错误。
- 手动确认底部导航和页面入口规划符合 MVP。

是否允许 commit：

- 允许。验证通过后使用中文 commit。

风险点：

- 默认 logs 页面是否删除需要谨慎，避免误删用户可能需要的示例参考。
- 页面路径一旦确定，后续 issues 会依赖该结构。

## Issue 2：首页 / 漫画书首页静态 UI

目标：

- 根据 01-home.png 实现首页 / 漫画书首页静态 UI。
- 展示默认漫画书、最近章节、免费次数、角色档案入口和我的入口。
- 使用 mock 章节状态：已完成、生成中、生成失败。

允许修改文件：

- manhua-xcx/pages/index/index.wxml
- manhua-xcx/pages/index/index.wxss
- manhua-xcx/pages/index/index.js
- manhua-xcx/pages/index/index.json
- mock 数据文件，若 Issue 1 已创建

禁止修改内容：

- 禁止实现真实接口请求。
- 禁止接真实 AI 任务状态。
- 禁止实现社区、评论、点赞、关注。
- 禁止实现支付或会员入口。

验证方式：

- 微信开发者工具预览首页无报错。
- 首页显示 mock 漫画书、章节列表和免费次数。
- 检查不存在非 MVP 入口。

是否允许 commit：

- 允许。验证通过后使用中文 commit。

风险点：

- 首页信息较多，需避免一次实现过多交互。
- 生成失败重试按钮本 issue 可只做静态展示，不实现流程。

## Issue 3：创建章节 / 日记输入页静态 UI

目标：

- 根据 02-create-chapter.png 和 03-diary-input.png 实现创建章节与填写日记照片的静态 UI。
- 支持 mock 表单状态、标签选择状态、照片占位列表。

允许修改文件：

- 新增或修改创建章节页面目录。
- 新增或修改日记输入页面目录。
- mock 数据文件。
- app.json 页面路径配置。

禁止修改内容：

- 禁止真实上传照片。
- 禁止真实提交生成任务。
- 禁止接内容安全接口。
- 禁止接 AI 服务。
- 禁止修改 PRD 中页数和照片数量规则。

验证方式：

- 页面可从首页或创建入口打开。
- 展示章节标题、日期、页数、标签、日记输入、照片网格和提交按钮。
- 照片数量文案保持 1-9 张。
- 页数范围不超出 PRD。

是否允许 commit：

- 允许。验证通过后使用中文 commit。

风险点：

- UI 图页数选项展示为随机、1、2、4、6、8，PRD 支持自定义 1-8；实现时需先按静态 UI 展示，不擅自改变产品规则。
- 拖动排序可先只显示提示，不实现复杂交互。

允许修改文件：

- 新增或修改角色档案页面目录。
- mock 数据文件。

禁止修改内容：

是否允许 commit：

- 新增或修改章节分享页面目录。

- 允许。验证通过后使用中文 commit。

禁止修改内容：

## 推荐开发顺序

1. Issue 1：小程序基础导航和页面骨架确认。
2. Issue 2：首页 / 漫画书首页静态 UI。
3. Issue 3：创建章节 / 日记输入页静态 UI。
4. Issue 4：角色档案页静态 UI。
5. Issue 5：生成等待页静态 UI。
6. Issue 6：章节详情页静态 UI。
7. Issue 7：章节私密分享页静态 UI。
8. Issue 8：我的页 / 免费次数展示静态 UI。
9. Issue 9：统一 mock 数据和占位状态。
10. Issue 10：后续接口联调预留。

若下一轮开始开发，建议先执行 Issue 1；如果只想先看到视觉效果，可从 Issue 2 首页静态 UI 开始，但仍需先确认页面骨架。
