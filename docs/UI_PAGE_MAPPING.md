# UI 页面映射

本文档基于 docs/ui/ 中的 UI 参考图、PRD 和 MVP 范围整理。它只用于后续微信小程序页面开发拆分，不代表本轮开发页面代码。

## 总体结论

- UI 图共 9 张，均可映射到 MVP 页面或 MVP 状态。
- 当前小程序骨架位于 manhua-xcx/，现有页面仍是微信开发者工具默认示例页：pages/index 和 pages/logs。
- UI 图整体符合“私人日记漫画本”定位。
- 未发现社区、评论、点赞、关注、公开广场设计。
- 未发现支付、充值、会员、订单设计。
- 09-Failure.png 属于免费次数不足状态，不是支付页。
- 初版前端先使用 mock 数据和占位状态，不接真实 AI 服务。

## 页面映射表

### 01-home.png

- 对应页面：首页 / 漫画书首页。
- 页面用途：展示默认漫画书、最近章节、免费次数、角色档案入口和我的/设置入口。
- 页面入口：小程序启动首页；底部导航“首页”；生成页返回漫画书。
- 核心模块：
  - 页面标题“我的漫画日记本”。
  - 问候气泡。
  - 默认漫画书卡片。
  - “写今天的漫画日记”主按钮。
  - 今日剩余免费生成次数。
  - 最近章节列表。
  - 章节状态：已完成、生成中、生成失败。
  - 角色档案入口。
  - 我的 / 设置入口。
  - 底部导航：首页、创建、我的。
- 需要的 mock 数据：
  - 用户昵称和问候语。
  - 默认漫画书名称、封面、私密标记。
  - freeQuotaRemaining。
  - chapterList：title、date、coverUrl、status。
- 后续需要的接口：
  - 获取当前用户信息。
  - 获取默认漫画书。
  - 获取最近章节列表。
  - 获取免费次数。
  - 查询生成任务状态。
- 是否属于 MVP：是。
- 是否存在超出 PRD 的设计内容：无。“全部章节”入口属于章节列表延展，仍在查看历史章节范围内。

### 02-create-chapter.png

- 对应页面：创建章节页。
- 页面用途：填写章节标题、选择日期、选择漫画页数、选择情绪/风格标签。
- 页面入口：首页主按钮；底部导航“创建”；章节失败后重新创建时可复用入口。
- 核心模块：
  - 章节标题输入。
  - 日期选择。
  - 同一天可记录多章提示。
  - 漫画页数选择：随机、1、2、4、6、8。
  - 情绪/风格标签：温馨、搞笑、治愈、童话、可爱、日常、开心、难过、纪念。
  - 下一步按钮。
- 需要的 mock 数据：
  - draftChapterTitle。
  - diaryDate。
  - pageMode、pageCount。
  - selectedTags。
  - tagOptions。
- 后续需要的接口：
  - 创建章节草稿。
  - 保存章节基础信息。
  - 获取标签配置。
- 是否属于 MVP：是。
- 是否存在超出 PRD 的设计内容：无。页数选项没有列出 3、5、7，但 PRD 范围为 1-8，自定义能力实现时需确认是否按 UI 固定选项还是提供完整选择。

### 03-diary-input.png

- 对应页面：上传照片与填写日记页。
- 页面用途：输入日记原文、上传 1-9 张照片、提交生成前检查。
- 页面入口：创建章节页点击“下一步：填写日记和照片”。
- 核心模块：
  - 日记输入框。
  - 字数统计。
  - 隐私提示。
  - 照片上传网格。
  - 照片删除、继续添加、拖动排序提示。
  - 生成前检查：日记已填写、照片已上传、内容合规。
  - “提交并开始生成”按钮。
- 需要的 mock 数据：
  - diaryText。
  - diaryTextMaxLength。
  - photoList：url、sortOrder。
  - validationState。
  - selectedChapterDraft。
- 后续需要的接口：
  - 上传照片。
  - 删除照片。
  - 保存日记内容。
  - 提交生成任务。
  - 敏感词或内容安全检查。
- 是否属于 MVP：是。
- 是否存在超出 PRD 的设计内容：无。拖动排序是合理交互增强，可在静态 UI 阶段仅做提示，不先实现复杂拖拽。

### 04-character-profile.png

- 对应页面：角色档案页。
- 页面用途：维护固定主角角色档案，用于提升章节之间的人物一致性。
- 页面入口：首页“角色档案”；我的页“角色档案”；首次使用引导。
- 核心模块：
  - 角色说明卡。
  - 主角昵称。
  - 身份 / 关系。
  - 年龄段。
  - 发型、发色、脸型。
  - 常见表情标签。
  - 常穿服装。
  - 标志性配饰。
  - 性格关键词标签。
  - 参考头像 / 参考照片。
  - 保存角色档案按钮。
- 需要的 mock 数据：
  - characterProfile。
  - ageRangeOptions。
  - expressionOptions。
  - personalityOptions。
  - referenceImages。
- 后续需要的接口：
  - 获取角色档案。
  - 保存角色档案。
  - 上传角色参考图。
  - 删除角色参考图。
- 是否属于 MVP：是。
- 是否存在超出 PRD 的设计内容：无。页面文案明确“不是复杂训练”，符合低风险角色档案方案。

### 05-generating.png

- 对应页面：漫画生成任务等待页。
- 页面用途：展示异步生成任务状态，让用户可返回漫画书等待。
- 页面入口：提交生成任务成功后进入；从章节列表点击生成中章节进入。
- 核心模块：
  - 页面标题“漫画生成中”。
  - 当前章节标题。
  - 生成步骤：排队中、生成中、整理漫画页。
  - 预计等待时间。
  - 返回漫画书按钮。
  - 轻提示。
- 需要的 mock 数据：
  - generationTask：taskId、chapterTitle、status、currentStep、estimatedMinutes。
  - pollingState。
- 后续需要的接口：
  - 查询生成任务状态。
  - 获取章节生成结果。
  - 任务失败原因查询。
- 是否属于 MVP：是。
- 是否存在超出 PRD 的设计内容：无。预计时间可先作为 mock 文案，不承诺真实精度。

### 06-chapter-detail.png

- 对应页面：章节详情页。
- 页面用途：展示某章生成完成后的漫画图片，并提供分享、重新生成、删除入口。
- 页面入口：首页最近章节；章节列表；生成成功后跳转。
- 核心模块：
  - 章节标题和日期。
  - 私密章节标记。
  - 漫画图片分镜列表。
  - 隐藏原始日记提示。
  - 分享章节按钮。
  - 重新生成按钮。
  - 删除章节按钮。
  - 二次确认提示。
- 需要的 mock 数据：
  - chapterDetail：title、date、privacyStatus。
  - comicImages：url、sortOrder。
  - actionState。
- 后续需要的接口：
  - 获取章节详情。
  - 获取章节漫画图片。
  - 创建分享链接。
  - 重新生成章节。
  - 删除章节。
- 是否属于 MVP：是。
- 是否存在超出 PRD 的设计内容：无。页面默认不展示原始日记，符合隐私规则。

### 07-share-chapter.png

- 对应页面：章节私密分享页。
- 页面用途：未登录访问者通过私密链接查看被授权分享的章节内容。
- 页面入口：分享链接；章节详情页创建分享后外部访问。
- 核心模块：
  - 私密漫画章节标题。
  - 章节标题和“仅链接可见”标记。
  - 分享漫画图片列表。
  - 隐私说明：仅展示授权章节，不包含其他章节与原始日记。
  - 打开小程序按钮。
  - 返回按钮。
- 需要的 mock 数据：
  - shareToken。
  - sharedChapter：title、images、ownerDisplayName。
  - shareVisibility。
- 后续需要的接口：
  - 根据分享 token 获取章节公开展示数据。
  - 校验分享链接状态。
  - 打开小程序跳转参数。
- 是否属于 MVP：是。
- 是否存在超出 PRD 的设计内容：无。按钮“打开小程序”属于分享页合理入口，不代表社区或公开页。

### 08-mine.png

  - 隐私说明入口。
  - 关于产品入口。
  - 基础设置入口。
  - 底部导航。
- 需要的 mock 数据：
  - userProfile。
  - defaultComicBook。
  - freeQuotaRemaining。
  - joinDays。
- 后续需要的接口：

  - 获取默认漫画书。
  - 获取免费次数。

- 是否属于 MVP：是。
- 是否存在超出 PRD 的设计内容：无。“基础设置”需保持轻量，不进入后台管理、支付或社区。

### 09-Failure.png

- 对应页面：免费次数不足提示页 / 生成次数不足状态。
- 页面用途：当免费次数不足时提示用户暂不可生成，并提供返回入口。
- 页面入口：提交生成前发现免费次数不足；我的页免费次数入口；首页免费次数提示。
- 核心模块：
  - 页面标题“暂时不能生成”。
  - 免费次数用完提示。
  - 当前剩余次数 0 次。
  - 初版免费次数说明。
  - 返回漫画书按钮。
  - 稍后再试按钮。
- 需要的 mock 数据：
  - freeQuotaRemaining。
  - quotaResetHint。
  - blockedReason。
- 后续需要的接口：
  - 获取免费次数。
  - 生成前校验免费次数。
- 是否属于 MVP：是。
- 是否存在超出 PRD 的设计内容：无。页面明确“不包含充值、会员或购买入口”，符合 PRD。

## 当前小程序骨架对应关系

- manhua-xcx/pages/index：当前默认示例首页，后续可在 Issue 1 后决定是否改造为首页 / 漫画书首页。
- manhua-xcx/pages/logs：当前默认示例日志页，MVP 不需要日志页，后续应在页面骨架确认时决定保留、删除或替换；删除需单独任务确认。

## 超出 MVP 风险清单

以下内容不应在页面实现中出现：

- 社区、广场、推荐流。
- 评论、点赞、关注。
- 整本书分享。
- 单张漫画图独立分享。
- 支付、充值、会员、订单、余额。
- 后台管理系统入口。
- 多漫画风格切换。
- 真实 AI 供应商接入。

## Mock 优先原则

首轮页面实现只允许使用 mock 数据和占位状态：

- 不接真实 AI 服务。
- 不调用 Java 后端接口。
- 不创建数据库。
- 不实现真实微信登录链路。
- 不实现真实上传存储。
- 不实现真实分享链接。
